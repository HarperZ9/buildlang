A compiler with receipts, not just a language pitch.

QuantaLang is an effects-oriented systems language built around quantac. The current adoption path is practical: compile and run real programs through the C backend, emit HLSL/GLSL shader source, and verify a small semantic corpus across backend work.

Current state

What is actually true now.

C backend is the adoption path

The working compiler core lowers source through lexer, parser, type checker, MIR, and C99 generation. The C backend is the only backend with end-to-end native execution verified by the compiler test suite.

HLSL and GLSL shader output are working

Shader examples compile to readable HLSL/GLSL from the same QuantaLang source. They are useful for graphics experiments and ReShade/Vulkan-style workflows without maintaining string-literal shader code.

Experimental means experimental

Rust, LLVM IR, WebAssembly, SPIR-V, x86-64, and ARM64 remain experimental. They are wired into quantac build --target, but they do not carry the same end-to-end promise as C today.

Showcase

What it can demonstrate.

QuantaLang is already useful as a compiler research surface: effects, typed systems syntax, shader math, a C execution route, corpus receipts, and editor/tooling integration.

01

Algebraic effects

Effect declarations, perform, handle, and explicit handler dispatch are part of the language surface and quickstart path.

02

Native execution through C99

quantac run compiles through generated C and a system C compiler, with public fixtures covering functions, loops, traits, dynamic dispatch, pattern matching, effects, and color/math examples.

03

Shader-oriented output

vignette_shader.quanta and other graphics fixtures show reusable math emitted to HLSL/GLSL for rendering experiments.

04

Receipts and diagnostics

quantac corpus verify checks the 8-program semantic corpus, C/Rust receipts, and real C-backend stdout. quantac doctor reports local backend readiness.

One source shape

CPU code and shader code live in the same language family.

The page now showcases the supported path: CPU examples run through C; shader examples emit HLSL/GLSL. The more speculative targets are visible, but plainly labelled as research-grade.

effects_greeting.quanta
effect Greeter {
  greet(name: str) -> str;
}

fn say_hello(name: str) -> str {
  perform Greeter.greet(name)
}

fn main() {
  handle say_hello("Alice") with Greeter {
    greet(name) => "Hello, " + name + "!"
  }
}
vignette_shader.quanta
#[fragment]
fn PS_Vignette(uv: vec2) -> vec4 {
  let color = tex2d(uv);
  let vig = vignette(color.x, color.y, 0.5, 0.6);
  vec4(color.x * vig, color.y * vig, color.z * vig, 1.0)
}

Backend map

Different targets, different maturity.

CWorking adoption target

C99 output, native execution, semantic-corpus C receipt, quickstart run path.

HLSLWorking shader output

Readable DirectX/ReShade-style shader source from graphics fixtures.

GLSLWorking shader output

OpenGL/Vulkan-style shader source for graphics experiments.

RustExperimental subset

Metadata and executable smoke slices over the semantic corpus; unsupported MIR errors loudly.

LLVM / WASM / SPIR-VExperimental research targets

Wired through CLI, useful for backend research, not the adoption promise.

x86-64 / ARM64Experimental assembly/object routes

Assembly/machine-code work exists, with linker/runtime integration still maturing.

Editor support

The VS Code extension provides syntax highlighting, brackets, comment toggles, and file icons. The CLI also exposes quantac lsp, quantac fmt, and quantac pkg, with LSP/package-manager maturity still documented as partial.

Quanta Universe

Quanta Universe is the broader alpha ecosystem: rendering, color, finance, systems, and tooling modules written in QuantaLang. It is a demonstration surface, not yet a whole-ecosystem build.
