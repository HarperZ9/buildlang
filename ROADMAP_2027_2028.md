**Historical planning record. Superseded 2026-06-15.**

This document is preserved as portfolio lineage, not as current BuildLang
execution posture. Current BuildLang posture is wind-down/preservation:
C-backed compiler core, capability/effect accountability, receipts, and public
claim accuracy stay maintained; Rust and x86/x64 remain experimental; broad
self-hosting and ecosystem expansion are not active lanes.

## Context

**Previous roadmap (M1-M8) is historical.** The current compiler is healthy in
its verified C-backed core, but the old end-to-end counts and tooling wiring
claims have been superseded by the 2026-06-15 status audit.

The question previously shifted to building with the compiler. Current posture
instead pauses broad expansion and preserves the verified compiler/accountability
surface.

## Track A: Compiler & Tooling (Language Maturity)

### A1: Type Checker - Method Calls on User Types
**Problem:** The type checker rejects `.method()` on user-defined structs. We work around this with free functions, but it blocks idiomatic BuildLang code. All 500K+ lines of ecosystem .bld code uses method syntax.
**Deliverable:** Update `types/infer.rs` to resolve method calls via impl blocks during type checking, not just codegen.
**Impact:** Unblocks ALL ecosystem .bld code from compiling.

### A2: Wire CLI Tooling
**Problem:** LSP (8.2K lines), Formatter (4.6K lines), Package Manager (5.6K lines) are complete but not CLI-accessible.
**Deliverable:** Wire `buildc lsp`, `buildc fmt`, `buildc pkg` subcommands. The code exists - just needs dispatch in main.rs.
**Impact:** Developer experience - IDE support, auto-formatting, dependency management.

### A3: Closure Structs (Escaping Closures)
**Problem:** Closures can't be returned from functions or stored in data structures. Lambda lifting only works for same-scope calls.
**Deliverable:** Generate closure structs (fn ptr + captured env) when a closure escapes its defining scope.
**Impact:** Enables functional patterns, iterator chains (.map/.filter/.collect), callbacks.

### A4: Trait Method Resolution in Type Checker
**Problem:** `impl Trait for Type` methods aren't visible to the type checker - only codegen sees them.
**Deliverable:** Register trait impls in the type context during type checking.
**Impact:** Enables trait-based polymorphism in .bld code.

### A5: String Improvements
**Problem:** String handling is limited - no slicing, no char iteration, no contains/starts_with/split.
**Deliverable:** Add string builtins to C runtime: `string_contains`, `string_starts_with`, `string_split`, `string_slice`, `string_char_at`.
**Impact:** Real text processing in BuildLang.

## Track B: Python Application Ports (5 Projects → BuildLang)

### B1: Port build-color (4.8K → ~3K BuildLang)
**The color science library - foundation for calibrate-pro.**
- 15+ color space conversions (sRGB ↔ Oklab ↔ JzAzBz ↔ Lab ↔ XYZ ↔ ICtCp)
- 12 tone mapping operators (ACES filmic, AgX, Reinhard, BT.2390)
- 7 delta-E metrics (CIE76, CIE94, CIEDE2000, CMC, JzAzBz, Oklab, HyAB)
- 9 chromatic adaptation methods (Bradford, CAT16, Von Kries)
- Spectral rendering (Planck blackbody, CIE 1931 CMFs, daylight illuminants)
**Port strategy:** All pure math - no numpy required for scalar operations.
**Proof point:** CIEDE2000 + all color space roundtrips match Python to 6 decimal places.

### B2: Port build-oracle (2.7K → ~2K BuildLang)
**Time series forecasting - the prediction brain.**
- ARIMA: Levinson-Durbin AR estimation, differencing, auto_arima
- PELT changepoint detection (dynamic programming)
- Classical decomposition (trend, seasonal, residual)
- Autodiff: Dual number forward-mode automatic differentiation
- Feature engineering: rolling stats, lags, differencing transforms
- Metrics: MAE, RMSE, MAPE, SMAPE, R²
**Port strategy:** Core loops are pure math. Skip neural.py (needs matrix ops) for now.
**Proof point:** ARIMA forecast on synthetic sine wave matches Python output. Autodiff computes f'(x) = 2x+cos(x) correctly.

### B3: Port build-finance (4.6K → ~1.5K BuildLang)
**Trading toolkit - indicators, risk, backtesting.**
- 10 technical indicators: SMA, EMA, RSI, MACD, Bollinger, ATR, Stochastic, VWAP, ADX, OBV
- 14 risk metrics: Sharpe, Sortino, Max Drawdown, VaR, CVaR, Calmar, Beta, Alpha
- 5 position sizing methods: Fixed, % Equity, Kelly, Volatility-Adjusted, Risk Parity
- Order book simulation with slippage/commission modeling
**Port strategy:** Indicators and risk are loop-based. Skip portfolio optimization (scipy-dependent).
**Proof point:** RSI + Bollinger Bands on test data match Python. Sharpe ratio correct to 4 decimals.

### B4: Port build-engine (1.1K → ~700 BuildLang)
**The orchestrator - connects oracle predictions to finance execution.**
- AdaptiveEngine feedback loop: predict → trade → evaluate → adjust weights
- PerformanceTracker: accuracy tracking per model, dynamic weight recalculation
- PredictionStrategy: ensemble voting across ARIMA/Prophet/Neural
- EngineConfig: all configuration as a struct
**Port strategy:** Pure state machine + data structures. Depends on B2 and B3 being done.
**Proof point:** Given mock predictions, produces correct weighted ensemble signal.

### B5: Port calibrate-pro/core (17K → ~5K BuildLang)
**The calibration engine itself - builds on B1.**
- Full color_math.py: all conversions + CIEDE2000 + Bradford (builds on build-color)
- lut_engine.py: 33³ LUT generation with tetrahedral interpolation, .cube format
- icc_profile.py: ICC v4 binary generation with VCGT tag
- aces.py: ACES RRT/ODT, ACEScg/ACEScc/ACEScct transforms
- Panel database: 58 characterized displays as HashMap
- Sensorless engine: EDID parsing, panel matching, profile estimation
**Port strategy:** Layer on top of B1. Largest port - do after B1-B4 are proven.
**Proof point:** Full pipeline: EDID → panel match → generate .cube + .icc → verify dE < 1.0.

### B6: Calibrate CLI Binary + Benchmark
- Native `calibrate` executable compiled from BuildLang via C backend + MSVC
- CLI: `calibrate --detect` / `calibrate --auto` / `calibrate --verify`
- HDR pipeline: PQ ST.2084, HLG, BT.2390 tone mapping
- Performance: benchmark 1000 LUT evaluations vs Python
**Proof point:** Native binary in <1 second vs 10+ seconds Python.

## Risk Items

| Risk | Impact | Mitigation |
|------|--------|------------|
| Type checker method resolution (A1) | Blocks all ecosystem compilation | Can stub methods as free functions initially |
| Massive .bld files (Photon 97K) | May hit performance/memory limits | Compile module-by-module, not monolithic |
| numpy-dependent Python code | Can't port vectorized array ops | Port scalar versions; skip scipy.optimize |
| Escaping closures (A3) | Complex refactor with cascading effects | Implement only for simple cases first |
| Five apps in parallel | Spread too thin | B1→B2→B3→B4 is sequential; B5-B6 after all others |
